# 公开演示模式设计

## 目标

为当前 Vercel 网站增加一个可随时关闭的公开演示模式。演示模式下不要求登录，访客打开网站即可使用；每次刷新或重新打开网页时清除上一轮浏览器数据。

## 配置方式

- 使用 `NEXT_PUBLIC_DEMO_MODE=true` 开启演示模式。
- 将该变量改为 `false` 或删除后，恢复现有登录流程和持久化行为。
- 当前 Vercel Production 环境开启该变量，本地和其他环境默认保持关闭。

## 用户体验

- 演示模式下使用固定的内存用户“演示访客”，不保存手机号。
- 受保护页面不再跳转到 `/login`。
- 直接访问 `/login` 时重定向到首页。
- 侧边栏显示“演示访客 / 数据刷新后清空”，不显示个人资料、退出登录或登录入口。
- 同一次页面打开期间，上传文稿、练习记录和报告可以正常生成并在页面间使用。
- 浏览器刷新或重新打开网站后，上一轮用户、记录、报告、文稿和练习状态全部清空。

## 架构与初始化顺序

新增一个独立的演示模式模块，集中提供：

- 是否开启演示模式的布尔值；
- 固定演示用户；
- 只在当前页面运行期存在的内存键值存储；
- 清除 `speakcoach_*` 浏览器存储和重置演示会话的函数。

数据层在演示模式下把用户、记录、报告和文稿读写切换到内存键值存储，不再读取或写入 `localStorage`。内存数据在同一次页面打开期间共享，浏览器刷新时自然归零。

`ClientInit` 在演示模式下先隐藏子页面，清除旧本地数据、重置内存存储和 Zustand 会话状态，然后设置固定演示用户，最后再渲染页面。这样记录页或报告页不会在清理前短暂读到旧数据。

普通模式继续使用现有 `initUser()` 恢复登录用户，不改变当前登录逻辑。

## 数据清理边界

每次演示模式页面首次加载时：

- 删除 `localStorage` 中所有键名以 `speakcoach_` 开头的历史数据，包括用户、记录、报告和 `speakcoach_file_*` 文稿；
- 清空当前页面运行期的演示内存存储；
- 重置内存中的模式选择、上传文件、当前练习、计时、暂停次数和转写内容；
- 通过 Next.js proxy 让浏览器删除旧的 `speakcoach_auth` 和 `speakcoach_otp` Cookie。

只清理本应用命名空间，不触碰同域名下其他存储键。

## 路由与认证

- `useRequireAuth` 在演示模式下直接返回演示用户并跳过登录重定向。
- Next.js proxy 在演示模式下将 `/login` 重定向到 `/`，同时清理旧认证 Cookie。
- 登录 API 保留不删，关闭演示模式后仍可恢复原流程。

## 错误处理

浏览器存储访问使用保护性检查；单个损坏或不可访问的存储项不能阻止网站进入演示模式。即使旧 `localStorage` 数据删除失败，演示模式的数据层也只读取新的内存存储，因此旧记录不会进入页面。

## 测试与发布

- 单元测试覆盖演示模式开关、内存存储生命周期、仅删除 `speakcoach_*` 键以及演示用户行为。
- 验证普通模式仍保持登录守卫。
- 执行 ESLint、TypeScript 和 Next.js 生产构建。
- 本地或预览环境验证：无登录跳转、侧边栏演示状态、创建数据后页面间可见、刷新后记录归零、`/login` 重定向。
- 将 `NEXT_PUBLIC_DEMO_MODE=true` 写入 Vercel Production 环境，重新部署并对固定生产网址执行同样的冒烟验证。

## 成功标准

- 新访客无需登录即可进入首页、记录页和练习流程。
- 页面中不再出现登录、退出登录或个人资料入口。
- 同一次页面加载期间产生的数据可用。
- 刷新页面后所有 SpeakCoach 浏览器数据清空。
- Vercel 固定生产网址继续返回正常页面和 API 响应。
